Overview

WebServ is a C++98 HTTP/1.1 compliant web server implementing non-blocking I/O with epoll for high performance.

Quick Start

Compilation

make

Running the Server

./webserv webserv.conf

Testing

curl http://localhost:8080/
curl -X POST -d "test=data" http://localhost:8080/cgi-bin/test.py
curl -X DELETE http://localhost:8080/uploads/test.txt

Features

  • HTTP/1.1 Protocol (RFC 9112)
  • GET, POST, DELETE methods
  • CGI Support (.py, .php, .pl, .rb, .sh)
  • File upload (multipart/form-data)
  • Directory listing (autoindex)
  • HTTP redirections (301/302)
  • Custom error pages
  • Multiple server instances
  • Non-blocking I/O with epoll

Configuration

WebServ uses NGINX-style configuration files:

server {
    listen 8080;
    server_name localhost;
    root ./www;
    index index.html;
    
    location / {
        allowed_methods GET POST;
    }
    
    location /cgi-bin {
        allowed_methods GET POST;
        cgi_extensions .py .php .pl .rb .sh;
    }
}

API Endpoints

GET Requests

Retrieve static files or execute CGI scripts:

GET /index.html
GET /cgi-bin/test.py?name=John&age=30

POST Requests

Submit data or upload files:

POST /upload
Content-Type: multipart/form-data

POST /cgi-bin/test.py
Content-Type: application/x-www-form-urlencoded

DELETE Requests

Remove files from the server:

DELETE /uploads/file.txt

Error Handling

WebServ provides custom error pages for common HTTP errors:

  • 400 - Bad Request
  • 403 - Forbidden
  • 404 - Not Found
  • 405 - Method Not Allowed
  • 500 - Internal Server Error

Performance

Tested with siege for concurrent connections:

siege -b -t 30s -c 100 http://localhost:8080/

Standards Compliance

  • C++98 Standard
  • HTTP/1.1 (RFC 9112)
  • CGI/1.1 Specification
  • POSIX Standards
